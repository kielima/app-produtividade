## 📋 Como usar este arquivo

### O que é um projeto (vs. tarefa)?
Um projeto tem **objetivo claro**, **múltiplas etapas** e um **resultado concreto**.
Ações avulsas de 1 passo ficam em `Minhas_Tarefas.md`.

### Estrutura de cada projeto
- [ ] **Nome** [Status] [Prioridade]
  - 🎯 **Objetivo:** o que se quer alcançar com este projeto
  - 📌 **Status atual:** onde o projeto está hoje
  - ➡️ **Próximos passos:** o que fazer primeiro para avançar

### Status dos projetos
- [A iniciar]    → Planejado, ainda não começou
- [Em andamento] → Ativamente em execução
- [Pausado]      → Temporariamente parado (inclua motivo se souber)
- [Concluído]    → Entregue ✅
- [Cancelado]    → Descartado definitivamente ❌

### Prioridade (impacto × viabilidade agora)
A prioridade responde: "Em quais projetos devo investir atenção nos próximos meses?"
É diferente do MoSCoW do Minhas_Tarefas.md, que responde "o que fazer agora no dia a dia".

- [P1] → Alto impacto + viável iniciar/continuar agora. Tem deadline real ou é base para outros projetos.
- [P2] → Alto impacto, mas depende de pré-requisito, recurso externo, terceiro ou horizonte de médio prazo.
- [P3] → Backlog frio: baixa urgência relativa, nice-to-have ou horizonte distante. Revisar a cada ciclo.

Revisar prioridades a cada lua nova ou mensalmente.

### Campos opcionais (após a linha de status/prioridade)
- 📅 yyyy-mm-dd       → Prazo ou meta de conclusão
- ⏳ Xh / Xd / Xmeses → Estimativa de duração
- 🔗 depende: X       → Dependência de outro projeto ou evento
- 💬 "nota"           → Qualquer contexto relevante

### Relação com outros arquivos
- Tarefas do dia a dia → `Minhas_Tarefas.md`
